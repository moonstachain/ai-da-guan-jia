# R6b：飞书妙搭仪表盘绑定指南

## 前提

- R6a 的 seed data 已生成
- 飞书妙搭应用已存在

## 步骤

### 1. 创建五张多维表

按 `dashboard/schemas/` 里的 5 个 JSON 文件，在飞书多维表中分别创建：

- 总控概览
- 组件热图
- 战略链路
- 组件责任
- 进化轨迹

### 2. 导入初始数据

把 `dashboard/seed/` 目录下的 JSON 文件导入对应的多维表。

### 3. 在妙搭应用中创建仪表盘页面

按 `dashboard/dashboard_spec.json` 的五区块结构：

- 区块 1：身份与协同（文本卡片）
- 区块 2：控制态（指标卡片，绑定总控概览）
- 区块 3：组件矩阵（表格视图，绑定组件热图）
- 区块 4：运行诊断（卡片组，绑定战略链路 + 进化轨迹）
- 区块 5：行动队列（筛选视图，绑定组件热图 + 组件责任）

### 4. 验证

- 打开妙搭应用，五个区块都有数据
- 组件热图的 12 个格子颜色与 `priority_band` 对应
- 进化轨迹显示 R1-R5 的五条记录
- 行动队列能筛出 `owner_gap` 和 `critical/high` 优先级

## 边界

- 卡片版式和颜色需要在妙搭中手动调整
- source view 的自动刷新需要后续配置 webhook
- 每张卡只回答一个管理问题，不混问题
