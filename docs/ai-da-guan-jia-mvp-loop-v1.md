# AI大管家最小 MVP 测试闭环

## Summary

第一轮不追求大而全，直接验证 `1 个真实小任务` 能不能顺畅闭环。

这一轮主要验证四件事：

- 共同治理者可以连续讲想法，而不被长时间思考打断
- `AI大管家` 能把连续表达收束成清晰任务
- 任务能推进到明确结果态
- 这一轮结束后，双方都能判断这套协同方式值不值得继续

## 任务范围

第一轮只选：

- 低风险
- 本地可完成
- 不需要登录 / 付款 / 发布

推荐任务类型：

- 写一份短文档
- 改一个小脚本
- 整理一个小流程
- 做一次本地信息汇总

第一轮不选：

- 需要外部账号登录的任务
- 涉及付款、发布、删除的任务
- 跨很多系统的大项目

## 对话协议

这一轮默认采用 `短回应收着听` 模式。

执行规则：

1. 共同治理者开始讲时，可以连续讲，不需要先整理
2. `AI大管家` 只做短确认，不进入长分析
3. 共同治理者说 `我讲完了` 后，再正式收束
4. 收束后至少给出：
   - 当前理解的目标
   - 明显约束
   - 预期结果
   - 准备如何推进

推荐触发语：

- 开始倾倒想法时：
  - `你先收着听`
  - `我先连续讲`
  - `你先别分析`
- 开始收束时：
  - `我讲完了`
  - `你总结一下`
  - `你开始拆`

## 最小闭环步骤

这一轮固定按下面顺序执行：

1. 共同治理者连续讲任务
2. `AI大管家` 短回应接住，不长思考
3. 共同治理者说 `我讲完了`
4. `AI大管家` 收束成明确任务定义
5. 如果信息足够，进入 `task-orchestrate` 思路推进
6. 如果信息稍缺但不影响推进，采用最小合理假设并明确说明
7. 如果碰到真实人类边界，只返回最小必要输入
8. 结束时给出一份短结果：
   - 做到了什么
   - 当前是什么状态
   - 如果还没闭环，只差什么

## 结果判定

这一轮只允许三种正式结果：

### `completed`

- 最小任务已经形成可接受闭环

### `blocked_needs_user`

- 只差共同治理者补一个明确输入

### `failed_partial`

- 当前状态和失败证据已保留
- 不要求共同治理者从头重讲

## Success Criteria

这一轮通过，至少要满足：

- 倾倒阶段没有被长时间打断
- 收束后的任务理解基本没有跑偏
- 返回的结果状态清楚
- 若需要人类输入，只提最小必要输入
- 共同治理者主观感觉比“来回问很多轮”更顺

如果这一轮不顺，问题归因只先落在这些类别：

- 接话太慢
- 过早分析
- 收束失真
- 打断过多
- 结果状态不清楚

## Assumptions And Defaults

- 第一轮只测 `最小真实任务`
- 默认优先验证协同节奏，不优先验证全部自动化能力
- 默认少打扰，但不会装作听懂却不收束
- 默认这一轮结束后，再决定是否扩展到更复杂任务
