# 原力OS 协同治理仪表盘 Prompt Pack v2

## 1. 产品定义

- 产品外显名：`原力OS`
- 页面名：`原力OS 协同治理总控舱`
- 目标：生成一个对陌生人也友好的单页治理驾驶舱
- 核心优先级：
  - 先让人看懂
  - 再让人更喜欢
  - 最后才考虑增加数据复杂度

这不是：

- 普通 BI 报表
- 企业经营数据大屏
- 康波历史仪表盘换皮
- 文档首页或产品介绍站

这是：

- `AI大管家` 的人类外显界面
- 一个帮助人类共同治理、共同判断、共同推进的系统操作界面

## 2. 使用方式

推荐投喂顺序：

1. 先贴 `主 Prompt`
2. 再贴 `第一屏导览原则`
3. 再贴 `数据绑定附录`
4. 再贴 `卡片映射附录`
5. 如果结果跑偏，再贴对应 `纠偏 Prompt`
6. 最后按 `验收清单` 复核

## 3. 主 Prompt

```text
你现在要设计并生成一个中文的高完成度单页驾驶舱，页面名称叫《原力OS 协同治理总控舱》。

请先理解，这一版的最高优先级不是“加更多卡片”，而是“让第一次看到这页的人也能迅速明白这是什么系统、怎么使用它”。

产品定义：
- 原力OS 是 AI大管家 的人类外显层。
- AI大管家 是系统内部治理引擎，负责统筹、推进、验真和闭环。
- 原力OS 是给人看的治理操作界面，负责让人理解系统当前状态、阻塞边界和下一步动作。
- 所以这不是普通 BI，不是项目管理列表，也不是概念海报。

页面目标：
- 让陌生用户一眼知道原力OS是什么
- 让协作者快速理解自己和系统的分工
- 让用户在一页内看懂：现在怎样、为什么这样、接下来做什么
- 页面必须同时具备：高级感、稳定感、可读性、治理导向

请把页面结构固定为四段，而不是直接堆数据：

第 0 段：Hero
- 标题：原力OS 协同治理总控舱
- 副标题：用一句话说明原力OS是 AI大管家 的人类外显层
- 展示最近刷新时间
- 展示当前运行态 badge
- 展示当前 frontstage focus badge

第 1 段：导览层 onboarding strip
- 用 3 个轻卡或 3 段短句完成认知引导
- 这 3 个模块必须分别回答：
  1. 原力OS 是什么
  2. 你和它怎么协同
  3. 这页该怎么看
- 这层必须短、口语、友好、先于数据出现
- 不能写成长文，不能像说明书页面

第 2 段：数据诊断层
- 保留 4 个 KPI
- 保留 4 张总览/诊断卡
- 标题文案更偏治理问题，不要太像字段标签

第 3 段：行动层
- 保留 4 张行动卡
- 必须让人一眼知道：我现在要处理什么、系统卡在哪里、下一步先推什么

阅读顺序必须清晰呈现：
1. 这是什么
2. 现在怎样
3. 为什么这样
4. 该做什么

视觉风格要求：
- 借鉴“300年康波周期分析仪表盘”的高级感、层次感和中心重力
- 但必须更克制、更安静、更像系统级产品
- 不做历史时间轴叙事
- 不做概念宣言式长文
- 不做泛企业 BI
- 不做科技感过强的炫技大屏

页面材质与风格：
- 深色分析舱风格，底色偏 slate / ink，不用纯黑
- 允许少量辉光、玻璃感、发光边框，但要克制
- 强调色固定三组：
  - cyan/blue = 正常推进 / 运行中
  - amber = 待你处理 / 人类边界
  - rose/red = 系统阻塞
- 文案要短、像系统提示，不像品牌宣言

页面中部应保留一个中心重力视觉，但不能做康波式历史时间轴。
请改成更适合治理 cockpit 的一种：
- 治理流态图
- 主线状态图
- 运行态脉冲图

非目标：
- 不要把白皮书整段塞进页面
- 不要做独立产品介绍页
- 不要默认系统已经支持复杂自动联动和 Feishu widget 精细布局
- 不要为了说明概念牺牲驾驶舱感

输出要求：
- 先给完整页面结构
- 再给 onboarding 设计
- 再给每张卡的布局和视觉建议
- 再给数据绑定建议
- 所有设计说明必须用中文
```

## 4. 第一屏导览原则

第一屏必须先帮助陌生用户进入系统，而不是直接扔给他数据卡。

这层导览必须满足：

- 比白皮书更短
- 比使用说明书更口语
- 比数据卡更先被看到
- 让用户 10 秒内理解“这是什么”

固定用 3 个模块：

### 模块 A：原力OS 是什么

必须表达清楚：

- 原力OS 是 AI大管家 的人类外显层
- 它是人机协同治理系统
- 它不是普通聊天助手，也不是普通报表

### 模块 B：你如何使用这页

必须表达清楚：

- 你负责目标、边界和不可替代判断
- 系统负责统筹、推进、验真和闭环
- 你不需要替系统拆每一步

### 模块 C：怎么读这页

必须表达清楚：

- 先看运行态
- 再看阻塞与原因
- 最后看待你处理和下一步动作

## 5. 原力OS 是什么

推荐给妙搭直接使用的短句素材：

- 原力OS 是 AI大管家 的人类外显层。
- 它把任务、线程、技能、来源、决策和写回组织成一个可被人理解和运营的治理系统。
- 它不是多聊一个助手，而是把“怎么协同、怎么判断、怎么闭环”做成一个系统界面。

## 6. 你如何使用这页

推荐给妙搭直接使用的短句素材：

- 你负责目标、边界和不可替代判断。
- 系统负责统筹、推进、验真和闭环。
- 当你想知道现在发生了什么，先看运行态。
- 当你想知道为什么停住了，看阻塞和诊断。
- 当你想知道该做什么，看待你处理和下一步动作。

## 9. 卡片映射附录

### Hero

- 大标题：`原力OS 协同治理总控舱`
- 副标题：明确原力OS与AI大管家的关系
- 元信息：
  - 最近刷新时间
  - 当前运行态
  - 当前 frontstage focus

### 导览层

固定三块：

1. `原力OS 是什么`
2. `你和它怎么协同`
3. `这页该怎么看`

### KPI 层

#### KPI-1 当前前台主线数
- 来源：`总控对象主表`
- 重点字段：`运营分段`、`对象状态`

#### KPI-2 blocked_needs_user 数
- 来源：`总控对象主表`、`任务总表`
- 重点字段：`需要人类输入`、`人类边界状态`

#### KPI-3 blocked_system 数
- 来源：`总控对象主表`
- 重点字段：`对象状态`、`阻塞原因`

#### KPI-4 数据源联通健康度
- 来源：`数据源同步表`
- 重点字段：`remote_reachable`、`github_backend`、`missing_sources_snapshot`

### 总览 / 诊断层

#### 卡 1：前台主线
- 来源：`总控对象主表`
- 推荐视图：`前台主线`

#### 卡 2：任务状态分布
- 来源：`任务总表`

#### 卡 3：数据源同步健康
- 来源：`数据源同步表`

#### 卡 4：最近治理推进
- 来源：
  - `Review批次表`
  - `决策记录表`
  - `治理动作与写回表`

### 行动层

#### 卡 1：待你处理
- 来源：`总控对象主表` 或 `任务总表`
- 过滤：`需要人类输入 != 空`

#### 卡 2：系统阻塞队列
- 来源：`总控对象主表`
- 过滤：`对象状态 = blocked_system`

#### 卡 3：下一步动作
- 来源：`总控对象主表`

#### 卡 4：能力与审批摩擦
- 来源：`技能与能力表`

## 10. 更像系统产品的视觉约束

这轮视觉优化方向不是“更花”，而是“更贵，但更安静”。

必须做到：

- Hero 更像系统首页，而不是营销页
- KPI 更像产品 summary，而不是传统报表数字块
- 导览层轻、薄、清楚，不抢数据层风头
- 页面中部有中心重力，但不做大叙事
- 通过排版、留白、材质感和色彩克制建立高级感

明确不要：

- 康波式时间轴主视觉
- 大段说明文案
- 浅色企业仪表盘模板
- 高饱和赛博风
- 过量发光、过量边框、过量动效

## 11. 纠偏 Prompt

### 11.1 如果生成结果“像文档首页，不像驾驶舱”

```text
当前结果过于像产品说明页或白皮书首页，而不是协同治理驾驶舱。

请做这些修正：
- 缩短说明性文字
- 强化 Hero、KPI、诊断、行动的层次
- 保留 onboarding，但必须轻量
- 让页面第一眼仍然像一个 operating cockpit
```

### 11.2 如果生成结果“像仪表盘，但不理解原力OS”

```text
当前结果像一个普通 dashboard，但还没接住“原力OS 是 AI大管家 的人类外显层”这个核心概念。

请修正：
- 在 Hero 和导览层明确写清原力OS与AI大管家的关系
- 让协同关系可见：人类负责目标与边界，系统负责推进与闭环
- 让页面更像治理系统，而不是项目列表或经营 BI
```

### 11.3 如果生成结果“已经易懂，但不够高级”

```text
当前页面已经比上一版更易懂，但视觉完成度还不够高。

请继续优化：
- 提升 Hero 的材质感和层次
- 提升 KPI 的精致度
- 强化中部主视觉的重力
- 保持克制，不要走炫技大屏路线
- 目标是达到接近“300年康波”那种完成度，但更安静、更系统化
```

## 12. 验收清单

- 陌生人第一眼先理解原力OS是什么，而不是先被数据卡淹没
- onboarding 层短、准、友好，不像说明书正文
- 页面仍然是单页总控舱，不是产品介绍站
- 阅读顺序清晰：
  - 这是什么
  - 现在怎样
  - 为什么这样
  - 该做什么
- 仍然保留：
  - Hero
  - 3 块导览
  - 4 KPI
  - 4 张诊断卡
  - 4 张行动卡
- 所有卡片仍能映射到 `协同治理` base 的真实表和真实字段
- 页面更易懂，也更高级，但没有滑向炫技大屏或康波叙事页
