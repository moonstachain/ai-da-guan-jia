# AutoClaw 自用版 Skill 冲突矩阵

这份矩阵只服务一个目标：让 `AI大管家` 在 AutoClaw 里长期可留存，而不是一装上就和预装 Skill 打架。

## 默认保留

| Skill | 角色 | 为什么保留 |
| --- | --- | --- |
| `AI大管家` | 前台总控 | 负责接任务、做判断、选知识源、准备收口 |
| `Deep Research` | 深度研究执行层 | 负责已经明确的问题研究，不负责总控分流 |
| `Browser Agent` | 网页执行层 | 负责浏览、点击、表单、抓取，不负责总控分流 |
| 图片搜索 / 图片生成 | 图片执行层 | 负责图片任务，不负责统筹与边界判断 |

## 建议弱化或关闭

| 冲突类型 | 典型症状 | 建议 |
| --- | --- | --- |
| 泛任务理解 Skill | 多个 Skill 都在重复改写任务 | 保留 `AI大管家`，关闭其余总控型任务分流 Skill |
| 泛研究型 Skill | 明明要路由，却直接开始长篇研究 | 保留 `Deep Research` 做执行层，别让它兼任前台总控 |
| 泛浏览型 Skill | 还没判断是否该浏览，就已经开始开网页 | 保留 `Browser Agent` 做执行层，入口判断交给 `AI大管家` |
| 全能代理型 Skill | 什么都想做，导致边界混乱 | 优先关闭；自用期先保持角色单一 |

## 默认调用顺序

1. 先叫 `AI大管家`
2. 由它判断这是不是研究、浏览、图片或知识源选择问题
3. 需要执行时，再切给对应下游 Skill
4. 收口前再回到 `AI大管家`

## 明确不让 `AI大管家` 抢的活

- 深度研究正文
- 网页浏览与操作
- 图片生成、搜图、修图
- 已经明确的单一执行任务

## 每周复查指标

- `被主动唤起次数`
- `误唤起次数`
- `该调用却没调用次数`
- `与其他 Skill 冲突次数`
- `一次任务中重复改写任务定义的次数`
