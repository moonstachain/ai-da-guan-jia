Use this fixed taxonomy. Do not invent new top-level categories during normal task execution.

- `research`

- `implement`
- `debug`
- `review`
- `sync`
- `publish`
- `governance`

- `skill`
- `code`

- `workflow`

## Labels

Always derive exactly four prefixed labels:

- `type:<value>`
- `domain:<value>`
- `state:<value>`
- `artifact:<value>`
