# agency-engineering — 全栈工程能力中枢

## 能力决策树

根据用户请求的关键词，自动路由到对应子能力：

### 路由规则

1. 涉及"API设计、数据库、微服务、服务端"→ **后端架构**模式
2. 涉及"页面、组件、UI、CSS、React、Vue"→ **前端开发**模式
3. 涉及"部署、CI/CD、Docker、监控、运维"→ **DevOps自动化**模式
4. 涉及"模型、训练、推理、embedding、向量"→ **AI工程**模式
5. 涉及"原型、MVP、快速验证、PoC"→ **快速原型**模式
6. 涉及复杂系统设计或架构评审→ **高级开发**模式（综合所有子能力）

### 后端架构模式

- 分析需求，输出 API 设计文档
- 选择技术栈（Python/Node/Go），说明理由
- 实现核心逻辑，包含错误处理和日志
- 提供数据库 schema 和迁移脚本

### 前端开发模式

- 分析设计需求或截图
- 选择框架（React/Vue/原生），说明理由
- 实现响应式布局 + 组件化结构
- 包含样式、交互和可访问性

### DevOps自动化模式

- 分析部署需求和现有基础设施
- 配置 CI/CD 管道（GitHub Actions/GitLab CI）
- 容器化（Dockerfile + docker-compose）
- 监控和告警配置

### AI工程模式

- 分析 AI/ML 需求
- 模型选择和集成方案
- 数据管道设计
- 推理服务部署

### 快速原型模式

- 30分钟内输出可运行的最小原型
- 优先使用最简技术栈
- 核心假设验证优先于代码质量
- 输出包含"下一步如果要生产化需要做什么"

## 通用规则

- 所有代码必须包含错误处理
- 提供运行和测试说明
- 说明技术选型理由
- 复杂任务先输出设计方案再写代码
