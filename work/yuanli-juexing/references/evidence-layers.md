# Evidence Layers

Use this file to keep `原力觉醒` grounded.

## Canonical Evidence Tags

Use only these five evidence tags in v1:

### `explicit_statement`

Use when the user directly says what they value, prefer, reject, or want.

Examples:

- "最关键的是关窍"
- "让 AI 找到你自己学习"
- "你的原力是一层层拆"

### `behavior_trace`

Use when the user's actions or task patterns imply a stable preference.

Examples:

- repeatedly asking for high autonomy and low interruption
- organizing work around leverage points instead of exhaustive lists
- feeding Codex long contextual materials to improve future collaboration

### `repeated_pattern`

Use when a signal appears across multiple sources or across time.

Examples:

- repeated `关窍` language
- repeated global-optimum framing
- repeated emphasis on AI orchestration over single-tool mastery

### `jung_inference`

Use only on the human-force side when applying Jung as an interpretive lens.

Examples:

- shadow pattern
- golden shadow pattern
- individuation tension
- integration task

Hard rule:

- never present `jung_inference` as medical fact or an absolute identity verdict

### `open_question`

Use when the evidence is too thin, contradictory, or awaiting user confirmation.

Examples:

- whether a strong drive is a core value or a compensation pattern
- whether a projected ideal belongs in `golden shadow` or explicit aspiration

## Confidence Bands

Every upward inference should carry one confidence band:

- `provisional`
- `repeated`
- `validated_by_user`

Do not silently upgrade confidence.

## Promotion Discipline

Only promote a signal when it changes collaboration behavior:

- raw clue -> evidence tag
- evidence tag -> force pattern
- force pattern -> bridge rule
- bridge rule -> collaboration protocol

If the higher layer does not change how Codex should work, keep it lower.

## Distortion Checks

Before finalizing a packet, ask:

1. Did I confuse rhetorical intensity with stable truth?
2. Did I elevate one emotionally charged moment into identity?
3. Did I use Jung to decorate weak evidence?
4. Did I write a flattering myth instead of a useful protocol?
5. Did I turn a human tension into a simplistic label?
