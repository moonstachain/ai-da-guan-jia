# 治理成熟度评估模型

## 十维度评分标准

每个维度 1-4 分。1=起步，2=可重复，3=已定义，4=持续优化。

---

### D1 知识治理（当前：3）

| 分数 | 标准 |
|------|------|
| 1 | 知识散落在聊天记录中，无结构化存储 |
| 2 | 有知识库但不完整，部分关键知识已结构化 |
| 3 | 知识分层明确（L0-L5），有 canonical source，有更新机制 |
| 4 | 知识图谱自动化更新，知识缺口自动识别，跨项目知识复用率>80% |

提分路径：信息溯源补全（TS-KB-07-EVIDENCE）→ 知识图谱自动化

### D2 任务治理（当前：3）

| 分数 | 标准 |
|------|------|

| 4 | Task Spec 自动生成+自动验收，任务全生命周期可追溯 |

提分路径：Task Spec 自动验收机制 → 全生命周期追溯
