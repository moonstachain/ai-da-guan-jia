# YSE 原力战略引擎详细流程

## 定位

为成长型企业提供"AI原生的战略诊断→模式创新→执行追踪→递归进化"全闭环服务。

## 四层引擎架构

```
第四层：递归进化引擎
  └─ 每轮结果反馈到下一轮优化
  └─ 月度Review + 季度深盘 + 年度战略校准

第三层：执行闭环引擎
  └─ Task Spec → Codex执行 → 飞书多维表 → 妙搭仪表盘
  └─ 关键路径：战略任务追踪表是闭环终点

第二层：战略生成引擎
  └─ 诊断 → 对标 → 建模 → 创新 → 路线图
  └─ 七步法详见下方

第一层：知识图谱引擎
  └─ 康波智库 + 模式创新库 + 对标库 + 客户数据湖
  └─ 每个客户沉淀的数据+模式让下一个客户更快更深
```

## 七步交付法

### Step 1：业务全景梳理
- 物料全丢进 Claude 知识库
- 用口语聊（"东北表妹聊天法"——颠过来倒过去掰开揉碎讲）
- 产出：业务全景图 + 关键假设清单

### Step 2：财务建模与UE分析
- 单元经济学拆解
- 关键公式：GMV / 客单价 / 佣金率 / BEP / 安全边际
- 产出：财务建模工作底稿（.xlsx）

### Step 3：全球对标研究
- 找 6-10 个全球对标企业
- 维度：商业模式 / 融资历程 / 关键指标 / 差异化壁垒
- 产出：对标矩阵

### Step 4：商业模式创新设计
- 从当前模式出发，设计进化路径
- 收入模型重构（从单一到矩阵）
- 产出：商业模式进化路线图

### Step 5：飞书多维表架构
- 设计表结构（主数据表 + 分析表 + 配置表）
- 定义字段类型和表间关系
- 产出：飞书多维表架构方案 + 建表 Task Spec

### Step 6：妙搭驾驶舱开发
- 基于飞书数据搭建可视化界面
- Glass Card 暗黑美学 + 响应式设计
- 产出：妙搭应用代码

### Step 7：递归进化机制建立
- 月度Review机制（每月10日）
- 仪表盘自动刷新
- 战略微调简报
- 产出：月度Review SOP

## 已验证能力（回旋镖局 MVP）

- ✅ 48小时内完成 McKinsey 级全量战略诊断
- ✅ 财务建模深度不亚于专业财务顾问
- ✅ 全球对标数小时完成（传统需数周）
- ✅ 商业模式创新有结构化框架和定量回报推演
- ✅ 飞书多维表作为"活的战略操作系统"交付
- ✅ 妙搭驾驶舱作为客户可见的战略界面

## 产品化路径

```
P0 完善案例 (2026Q1-Q2): 回旋镖局月度Review + 季度深盘
P1 标准化 (2026Q2-Q3): 七步法封装为可复用的战略 Skill
P2 第二客户 (2026Q3): 验证可复制性
P3 产品化 (2026Q4): YSE 1.0 发布
```

## 客户数据隔离原则

- 每个客户一个独立的飞书 Base
- 客户数据不与内部治理数据混用
- 客户 Base 的 app_token 和 API 凭证单独管理
- 对标数据可跨客户复用，业务数据不可
